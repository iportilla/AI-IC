# Design Note Examples

These short examples show what it means to write a design note before asking AI to generate code.

Each note is intentionally lightweight. The goal is not heavy documentation. The goal is to force enough clarity on scope, inputs, outputs, and constraints that the AI can be guided well and the student can review the result responsibly.

## Quick Template

Use this structure for small assignments:

- **Problem**: What needs to be built?
- **Scope**: What is included and excluded?
- **Inputs**: What data comes in?
- **Outputs**: What should the program return, print, or save?
- **Constraints**: What rules must the implementation follow?
- **Validation**: How will you know it works?
